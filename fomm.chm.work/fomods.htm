fomods
[Previous] [Main] [Next]

fomod's are plain .zip files. In the absense of any fancy scripting, they should be structured so that extracting them into fallout's data folder installs them correctly. Any fomod specific data goes into a folder called 'fomod'.

Files which fomm expects to find in here are 'info.xml', which contains the mod name, version, author, etc. 'screenshot.jpg' (or .bmp, or .png) which contains a descriptive screenshot of the mod, and 'script.cs' which contains the mod installation script. Any of these files can be missing without causing problems, which allows fomm to install completely normal zip's with no additional work as long as they're structured correctly.

Scripting is done in C#. If you provide a script, fomm won't automatically install anything from the fomod; the script must install whatever it wants to itself. (See the Scripting examples chapter for some examples of scripts.)

Readme's should be in the root of the zip file, and be named 'readme - <name of the fomod without the file extension>.txt' (or .rtf, or .html). i.e. The readme of a fomod called 'test.fomod' would be called 'readme - test.txt'

Fomods by default are located in the subdirectory 'mods' in whichever directory fomm.exe exists in. There's no option inside fomm to change that, but you can pick a new directory by manually editing 'fomm\settings.xml'. Add a new xml node '<strValue name="FomodDir">your new path goes here</strValue>'.
