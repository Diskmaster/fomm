Command line options [Previous] [Main] [Next]

Fallout mod managers startup behaviour can be modified through the use of command line arguments. Primarily, specifing the name of a fomod, .fomod.zip, bsa or sdp as the first command line argument will open it with the appropriate utility. Other options are available, some of which the automatic installer can generate seperate shortcuts for. The full list is:

-help
Writes out this list to the console then exists

-setup
Opens a menu that allows you to change some options that are unavailable from within fomm.

-bsa-unpacker
Opens the bsa browser/unpacker utility

-bsa-creator
Opens the bsa creator utility

-sdp-editor
Opens the shader editor

-tessnip
Opens tessnip

-package-manager
Opens the package manager, as well as the main form

-install-tweaker
Opens the install tweaker utility

-no-uac-check
Skips the vista UAC check. Use this if fomm incorrectly detects UAC interference, or if you want fomm to end up installing data files into your virtual store for whatever reason. You can also add this option if you know fomm will never have permissions issues to make startup a tiny bit faster.

-mono
Disables features which are known to cause issues with mono. (Currently limited to syntax highlighting when editing scripts)
