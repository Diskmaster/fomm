void PerformBasicInstall()

Calling this function will extract all files from the package except for those in the fomod directory, and then activate all esps and esms. This is exactly equivelent to what fomm would do if no script was present at all.

int[] Select(string[] items, string[] previews, string[] descs, string title, bool many)
int[] Select(SelectOption[] options, string title, bool many)

Display a dialog that lets the user pick one or more options out of a selection. You can provide each selection with an attached screenshot and/or description. In the first version of this function, 'previews' is an array of paths to image files inside the fomod. Both the previews and descs arrays are allowed to be null, or to contain null entries. If the array isn't null, it must be the same length as the items array. In the second version, you pass an array of SelectOption structs, which is a custom struct given below.

struct SelectOption {
public string Item, Preview, Desc;

public SelectOption(string item, string preview, string desc);
}

The return value is an array of indicies of the selected items, or null if an error occured.

string[] GetAllPlugins()
string[] GetActivePlugins()
void SetPluginActivation(string s, bool activate)
void SetLoadOrder(int[] plugins)
void SetLoadOrder(int[] plugins, int position)
bool IsLoadOrderAutoSorted()
int GetAutoInsertionPoint(string plugin)
void AutoSortPlugins(string[] plugins)

Used to manipulate currently installed plugins. GetActivePlugins returns a load order sorted list of the currently active esps, and GetAllPlugins gets a list of all esms and esps in the data directory whether or not they are active. SetPluginActivation is used to set esp's as active or inactive. (Note that newely installed esps are initially inactive) The first load order function can be used to set the order of the entire plugin list at once, and the second performs the same as a drag drop on the load order menu. (i.e. specify a list of plugins you want moved, and the position in the list to insert them into.) IsLoadOrderSorted returns true if the current load order matches up with what the auto sort button on the plugin screen would set, or false otherwise. GetAutoInsertionPoint returns the position in the current load order that the auto sort would like to place a plugin with the specified name. AutoSortPlugins feeds a set of plugin names into the auto sorter, and sorts the array according to the result.

Used to replace a single shader in an sdp package. The shader name is case sensitive. The data must be an assembled pso or vso object. (i.e. the contents of one of the files you get by clicking 'export binary' in the shader editor.)

Form CreateCustomForm()

Creates a new form, which you can add controls to normally to construct a custom dialog. Use this rather than 'new Form()' to avoid the unknown zone security warning.

string GetLastError()

When an error occurs that doesn't raise an exception but sill causes the scripting function to fail, you can usually call GetLastError to get a string that gives a reason why the function call failed.

void SetupScriptCompiler(Plugin[] plugins)
void CompileResultScript(SubRecord sr, out Record r2, out string msg)
void CompileScript(Record r2, out string msg)

These functions are used to access fomm's script compiler; use them if you perform any tasks using the tessnip interface that requires scripts to be recompiled. SetupScriptCompiler must be called again for each new plugin you wish to compile scripts for. The 'plugins' array should contain a list of masters followed by the plugin in the order specified in the plugin header to ensure the compiler gets formIDs correct.

bool IsAIActive()

Simply returns true if ArchiveInvalidation Invalidated is installed or archiveinvalidation has been toggled on in fomm, or returns false otherwise. It will not return true if archiveinvalidation.txt is present, as that is not a reliable solution to the invalidation problem.

public enum TextureFormat {
R8G8B8=20,
A8R8G8B8=21,
X8R8G8B8=22,
DXT1=('D'<<0) | ('X'<<8) | ('T'<<16) | ('1'<<24),
DXT3=('D'<<0) | ('X'<<8) | ('T'<<16) | ('3'<<24),
DXT5=('D'<<0) | ('X'<<8) | ('T'<<16) | ('5'<<24),
}

These functions are used to manipulate textures. A texture must first be loaded or a blank texture created using LoadTexture or CreateTexture. You can then manually manipulate the data, which will always be A8R8G8B8 format regardless of the source, or copy between different textures. ReleaseTexture only needs to be used if you're generating a sufficient number of textures to fill memory; all textures will be released when the script finishes executing anyway. All saved textures are in dds format, although other types of textures can be loaded. Textures will always be loaded without mipmaps, but you have the option of automatically generating a new set when you save.
